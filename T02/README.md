# 🔄 T02: Control de versions – Treballant amb Git

## 📌 Breu descripció

Fins ara hem gestionat el control de versions directament des de la web de GitHub. Tot i que ens ha ajudat a resoldre molts problemes, aquesta metodologia té algunes limitacions importants:

- Lentitud a l’hora d’editar.
- L’editor web no és tan versàtil com un editor local (Visual Studio Code, Ghostwriter, etc.).
- Gestió poc eficient d’arxius i carpetes dins del repositori.

Per aquest motiu, començarem a treballar com es fa en el món professional: combinant **Git en local** amb un **repositori remot a GitHub**.

---

## 🧠 Context

Git és un sistema de control de versions descentralitzat creat per **Linus Torvalds** (creador de Linux). Va aparèixer abans que GitHub i va revolucionar el model centralitzat que s’utilitzava fins aquell moment.

GitHub és una plataforma que permet allotjar repositoris Git al núvol, però no és l’única alternativa (també existeixen GitLab, Bitbucket, etc.).

---

## ⚙️ Com treballarem

El flux de treball serà el següent:

1. Partirem d’un repositori creat a **GitHub**.
2. En farem una còpia sincronitzada en local (al nostre PC).
3. Els canvis es faran sempre en local.
4. Seguirem el flux habitual:
   - `add`
   - `commit`
5. Quan acabem de treballar, pujarem els canvis amb un `push`.
6. Quan tornem a treballar (especialment si canviem d’ordinador), farem un `pull` per assegurar-nos que tenim la versió actualitzada.

Això ens permet treballar de manera més eficient, professional i segura.
